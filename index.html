Pull request checks are not enough

Most tools run checks and show results. They rely on humans to enforce decisions. This leads to inconsistency, missed issues, and broken standards.

Manthan OS enforces decisions

Every pull request is evaluated using rules. Failures automatically block merges. No ambiguity. No drift.

Example

Rule: PR must include test updates

Pull Request: src/api.js changed, no tests

Result: FAIL

Reason: No test files modified

Outcome: Merge blocked until fixed

How it works

Pull Request -> Webhook -> Manthan -> Decision -> Status Check -> Blocked / Allowed

Manthan evaluates pull requests using contracts and enforces results via GitHub status checks.

Define rules as contracts

{ "name": "require-tests", "rule": "test_files_changed == true", "onFail": "reject" }

Features

Deterministic decisions: Same input -> same output
